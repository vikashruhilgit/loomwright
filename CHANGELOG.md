# Changelog

Full release history for the AI Agent Manager plugin. The two most recent entries are also summarized at the top of `CLAUDE.md`; this file is the complete record.

> Entries are historical: the version and count claims in each describe **that** release, not the current state. The authoritative current version/counts live in `plugin.json`, `hooks.json`, and the `agents/`/`commands/`/`skills/` directories.

**v14.19.0 — System Twin M2b slice 1a: an advisory ground-truth execution step in Supervisor Phase 4.5:** Lands the **ground-truth execution muscle** slice of milestone M2b (Pillar 2, "provable-done") — **advisory, additive, and with zero counted artifacts.** **(1) `scripts/run-ground-truth.sh`** — a deterministic, fail-safe runner that resolves a project's **declared executable-acceptance checks** and runs them, reporting a hard pass/fail on real behavior. The declaration source is a brief's `## Executable Acceptance` section (when present) or, as a fallback, a `.supervisor/twin/ground-truth.json` file; each declared check carries a `kind`: **`cmd:`** (an arbitrary shell command — exit 0 = pass, non-0 = fail), **`corpus-task:`** (run a single `eval-corpus/<task-id>/check.sh` task), or **`qa-executor:`** (recognized but **DEFERRED to slice 1b** — recorded as a per-check `unverified` with reason `qa_executor_dispatch_deferred_m2b_1b`, never run in this slice). The script prints a human/grep per-check block + a `Checks passed: M/N` line and exactly ONE machine-readable `GROUND_TRUTH_JSON: {...}` line (jq-built for injection safety). It is **fail-safe** — no declaration found emits `status:"skipped"` and absent `jq` emits `status:"unverified"` (both with `checks_total 0`) and the script ALWAYS exits 0 (mirroring `run-benchmark.sh` / `run-eval.sh`). **(2) `scripts/test-run-ground-truth.sh`** self-test (11 assertions: cmd pass/fail, corpus-task resolution, qa-executor deferral with ran:false, no-declaration skipped, no-jq unverified, fail-safe exit-0, leading-dash target preservation, exact-heading match, GROUND_TRUTH_JSON shape). **(3) Supervisor Phase 4.5 wiring** — Phase 4.5 invokes the runner **after** the Code Reviewer self-heal loop and folds the result into a new additive **`ground_truth`** SUPERVISOR_RESULT object plus flat `session_end` JSONL hard-signal fields. The signal is **advisory only**: it NEVER changes `heal_decision`, NEVER triggers a fix, and NEVER blocks the PR — exactly the contract-conformance / benchmark precedent. **(4) New `GROUND_TRUTH_JSON` schema** (`schema_version 1`; fields `ran` / `status: pass|advisory_failures|unverified|skipped` / `checks_total` / `checks_passed` / `pass_rate` "M/N" / `per_check[]` of `{kind, target, status: pass|fail|unverified, reason?}` / `commit` / `date`) and the `## Executable Acceptance` brief convention documented in `docs/RESULT_SCHEMAS.md`; SUPERVISOR_RESULT stays `schema_version 1` (same additive precedent as `contract_conformance` / `benchmark_result` / `branch_base` / `pr_state`). The ground-truth runner is **DISTINCT from** the eval harness (`run-eval.sh` / `EVAL_RESULT`, output-quality over a fixed corpus) and the canary benchmark (`run-benchmark.sh` / `BENCHMARK_JSON`, the hard-signal pipeline) — kept separate by name and directory. **Deferred:** M2b slice 1b (auto-dispatch the QA Executor's Playwright/app-execution muscle for web-app repos) and M2b part-2 (auto-running the full Launch Pad→Supervisor agent loop in CI against the corpus). **No new agent / command / hook / skill (still 14 / 16 / 51 / 19); the new scripts are uncounted by the doc-currency gate; additive schema (SUPERVISOR_RESULT stays `schema_version` 1).** Additive on top of v14.18.0.

**v14.8.0 — Phase 1.5 PRE-FLIGHT SYNC: remote-state reconciliation before any worker spawns:** Adds a mandatory **Phase 1.5 PRE-FLIGHT SYNC** gate to the Supervisor, placed *after* Phase 1 ACQUIRE (which already fetches/pulls the base so the feature branch starts fresh) and *before* Phase 2 PLAN spawns the Orchestrator or any worker — so overlapping or already-landed work is surfaced *before* tokens are spent on decomposition/execution. The gate fetches remote state, inspects recent `origin/$BASE_BRANCH` commits and open PRs, derives the canonical version + base-tip SHA, and classifies the requested work as **CLEAR / OVERLAP / SUPERSEDED** on two required signals: **(a) same-file overlap** — a recent commit or open PR whose changed files intersect the task's anticipated file set (the job brief's File Impact Map when present, else the task description); and **(b) already-merged equivalent** — recent `origin/$BASE_BRANCH` history that already implements the requested work (the SUPERSEDED case behind the v13.1.0→v14.0.0 stale-branch incident). **CLEAR is silent** (records a one-line pre-flight summary via Context-Keeper and proceeds). **OVERLAP/SUPERSEDED soft-gates interactively** via an `AskUserQuestion` (proceed-anyway / revise-scope / abort) that cites the specific commit SHAs / PR numbers **and** the intersecting paths — mirroring Launch Pad's Phase 2.5 feasibility gate — and **fails closed** under `--non-interactive` / non-TTY by aborting with `SUPERVISOR_RESULT.status: failed` + `error: "preflight_overlap_detected"` (surfaced by `/autonomous` as `AUTONOMOUS_RUN.status_reason: "preflight_overlap_detected"`, a terminal outcome the loop does not re-iterate). The gate is **stacked-iteration safe** (scopes the overlap scan to `$BASE_BRANCH` and never flags the parent iteration's own commits/PR), **degrades gracefully** on `gh`/`git` failure (records "pre-flight unverified", one warning, `preflight_sync = unverified`, continues — never hard-blocks on tooling), and is **bounded** (≤6 tool calls, ~20s/invocation ceiling; reuses Phase 1's fetch). `--skip-preflight-sync` short-circuits the gate as a deliberate Context-Keeper-recorded choice (it is a **flag, not a command**). Schema-wise: adds the optional additive **`preflight_sync`** field (`enum [clear, overlap_proceed, superseded_proceed, skipped, unverified] | null`) to SUPERVISOR_RESULT — `schema_version` stays **1**, same additive precedent as `branch_base` / `pr_state`, hook does not enumerate it — and the closed **`preflight_overlap_detected`** value to the AUTONOMOUS_RUN `status_reason` enum (`schema_version` stays **2**). **No new agent / command / skill / hook (still 13 / 14 / 50 / 19).** Additive on top of v14.7.0.
