# Changelog

Full release history for the AI Agent Manager plugin. The two most recent entries are also summarized at the top of `CLAUDE.md`; this file is the complete record.

> Entries are historical: the version and count claims in each describe **that** release, not the current state. The authoritative current version/counts live in `plugin.json`, `hooks.json`, and the `agents/`/`commands/`/`skills/` directories.

**v14.11.0 — `/obsidian`: a Full Linked Obsidian Vault projection of `.supervisor/`:** Adds a new read-only, opt-in slash command `/obsidian` backed by `scripts/build-vault.sh` (self-tested by `scripts/test-build-vault.sh`; neither counted by the doc-currency gate) that projects the local `.supervisor/{twin,logs,memory}` artifact trees into a configurable shared destination as a **Full Linked Obsidian Vault** at `<vault>/<slug>/` — cross-linked markdown notes a human can browse, search, and graph in Obsidian. The projection is **read-only** (it never mutates `.supervisor/` or any source of truth) and **content-hash idempotent** (re-running with unchanged inputs produces no rewrites, so the vault's modification times stay stable), **sparse-tolerant** (an absent `twin/`, `logs/`, or `memory/` subtree degrades gracefully rather than erroring), and **decoupled** — no agent ever reads the vault back, so a projection can never feed stale state into a run. The destination directory (and `<slug>`) are configurable so multiple projects can share one Obsidian vault without collision. This is **+1 command (14 → 15)**; **no new agent / skill / hook (still 13 / 15 / 50 / 19)** and **no schema change**. Additive on top of v14.10.0.

**v14.10.0 — System Twin foundation slice (advisory, propose-only, 3-pillar vertical):** Lands the first thin, additive, reversible vertical of the System Twin product direction (seeded `proposed` in v14.9.0's `CAPABILITY_BASELINE.json`) — one slice that exercises all three pillars on the plugin's own repo while staying **advisory and strictly subordinate to `CLAUDE.md`**, propose-only, and reversible. **Foundation (ST1):** introduces a `.supervisor/twin/` per-subsystem **System Contract** artifact store written **exclusively** by the repo-root sole writer `scripts/write-system-contract.sh` (worktree-guard refusing any git-worktree CWD, hash-chained provenance, atomic write), read via `scripts/read-system-contract.sh`, and self-tested by `scripts/test-system-contract.sh`. Adds a new **`SYSTEM_CONTRACT`** result schema plus additive **`SUPERVISOR_RESULT.contract_conformance`** / **`.benchmark_result`** fields and flat `session_end` JSONL hard-signal fields in `RESULT_SCHEMAS.md` (**`schema_version` stays 1**, same additive precedent as prior optional fields), and documents the Twin homing contract in `ARCHITECTURE_CONTRACTS.md`. Context-Keeper is deliberately **NOT** in this write path. **Pillar 1 — Predict (ST2):** Launch Pad Phase 3 ANALYZE reads the contract's dependency graph to produce an advisory **blast-radius / impact prediction**, degrading gracefully (no prediction, no error) when no contract exists. **Pillar 2 — Prove (ST3):** Supervisor Phase 4.5 runs an **advisory contract-conformance check** against the integrated diff (`git diff origin/main...HEAD`) plus a **deterministic benchmark** (`scripts/run-benchmark.sh` + corpus, self-tested by `scripts/test-benchmark.sh`); the SELF_HEAL completion tail spawns an **ephemeral builder** Task that writes/refreshes contracts via the sole writer from the pinned repo-root CWD, and emits a hard signal to **both** `SUPERVISOR_RESULT` and the `session_end` JSONL record. **Pillar 3 — Compound (ST4):** `scripts/build-insights.sh` aggregates the per-session hard signal into a trend, `/insights` surfaces it, `/dreaming` reads contract drift as a DISTILL input, and the rubric-grader **REPORTS** conformance / benchmark results as advisory lines that **never gate** the PR. **Guardrails throughout:** propose-only (no self-applied Twin writes without the existing human gate), advisory-subordinate-to-`CLAUDE.md`, sole-writer + pinned-CWD enforcement, and every new script self-tested. **Sibling-bug fix (found during this PR's review):** the existing `scripts/write-project-memory.sh` shared the same write-time-eviction provenance-chain break — the eviction entry's `prev_hash` hashed the line *with* its trailing newline while the reader hashes *without* it, so once the 200-entry cap was crossed the chain broke and `read-project-memory.sh` silently dropped every surviving fact; corrected to match the add-path/reader, with a new post-eviction read-back assertion added to `test-project-memory.sh`. The foundation adds **ZERO** counted artifacts — the builder is an ephemeral Task and the new helper scripts are not counted by the doc-currency gate. **No new agent / command / skill / hook (still 13 / 14 / 50 / 19). Additive on top of v14.9.0.**

**v14.9.0 — `/capability-check --strategy`: a grounded product-direction strategist mode:** Extends the v14.6.0 `/capability-check` command with a `--strategy` mode that converts a pure platform-adoption diff into a **grounded product-evolution pass**. The mode reuses the existing brainstorming skill's 5-lens scored-debate engine to generate, debate, and rank candidate **product DIRECTIONS** — a deliverable distinct from the default mode's CANDIDATE adoption report — scoring each on a **moat × feasibility × effort** rubric and ranking the survivors. A hard **grounding mandate** keeps the output tethered: every surfaced direction must cite **≥1 concrete product asset AND ≥1 newly-feasible enabler (or an explicit drop)**, so speculative or untethered ideas are filtered out before they reach the report; when nothing clears the bar the mode emits a **single one-line suppression** and exits rather than padding. Candidates are **deduped** against a new `product_directions` section seeded into `ai-agent-manager-plugin/docs/CAPABILITY_BASELINE.json` (initial entry: **System Twin = proposed**, three pillars — predict-before-touch / provable-done / compounding-expertise). The default (no-flag) `/capability-check` behavior is **guaranteed unchanged**. The mode is **propose-only** (it never self-applies or edits plugin code), **bounded** by `--max-fetches`, and the only write path is the explicit maintainer action `--update-baseline`, which records a direction's status (e.g. proposed → adopted/deferred) and nothing else. **No new agent / command / skill / hook (still 13 / 14 / 50 / 19).** Additive on top of v14.8.0.

**v14.8.0 — Phase 1.5 PRE-FLIGHT SYNC: remote-state reconciliation before any worker spawns:** Adds a mandatory **Phase 1.5 PRE-FLIGHT SYNC** gate to the Supervisor, placed *after* Phase 1 ACQUIRE (which already fetches/pulls the base so the feature branch starts fresh) and *before* Phase 2 PLAN spawns the Orchestrator or any worker — so overlapping or already-landed work is surfaced *before* tokens are spent on decomposition/execution. The gate fetches remote state, inspects recent `origin/$BASE_BRANCH` commits and open PRs, derives the canonical version + base-tip SHA, and classifies the requested work as **CLEAR / OVERLAP / SUPERSEDED** on two required signals: **(a) same-file overlap** — a recent commit or open PR whose changed files intersect the task's anticipated file set (the job brief's File Impact Map when present, else the task description); and **(b) already-merged equivalent** — recent `origin/$BASE_BRANCH` history that already implements the requested work (the SUPERSEDED case behind the v13.1.0→v14.0.0 stale-branch incident). **CLEAR is silent** (records a one-line pre-flight summary via Context-Keeper and proceeds). **OVERLAP/SUPERSEDED soft-gates interactively** via an `AskUserQuestion` (proceed-anyway / revise-scope / abort) that cites the specific commit SHAs / PR numbers **and** the intersecting paths — mirroring Launch Pad's Phase 2.5 feasibility gate — and **fails closed** under `--non-interactive` / non-TTY by aborting with `SUPERVISOR_RESULT.status: failed` + `error: "preflight_overlap_detected"` (surfaced by `/autonomous` as `AUTONOMOUS_RUN.status_reason: "preflight_overlap_detected"`, a terminal outcome the loop does not re-iterate). The gate is **stacked-iteration safe** (scopes the overlap scan to `$BASE_BRANCH` and never flags the parent iteration's own commits/PR), **degrades gracefully** on `gh`/`git` failure (records "pre-flight unverified", one warning, `preflight_sync = unverified`, continues — never hard-blocks on tooling), and is **bounded** (≤6 tool calls, ~20s/invocation ceiling; reuses Phase 1's fetch). `--skip-preflight-sync` short-circuits the gate as a deliberate Context-Keeper-recorded choice (it is a **flag, not a command**). Schema-wise: adds the optional additive **`preflight_sync`** field (`enum [clear, overlap_proceed, superseded_proceed, skipped, unverified] | null`) to SUPERVISOR_RESULT — `schema_version` stays **1**, same additive precedent as `branch_base` / `pr_state`, hook does not enumerate it — and the closed **`preflight_overlap_detected`** value to the AUTONOMOUS_RUN `status_reason` enum (`schema_version` stays **2**). **No new agent / command / skill / hook (still 13 / 14 / 50 / 19).** Additive on top of v14.7.0.

**v14.7.0 — `/insights`: local, Obsidian-friendly insights dashboard from session logs:** Adds a new slash command `/insights` that generates a local markdown **insights dashboard** (`.supervisor/insights/dashboard.md` plus per-run notes carrying Dataview-compatible frontmatter) from `.supervisor/logs/*.jsonl`, covering **work / quality / session-performance** — completion rate, self-heal outcomes, rubric scores, subtask counts, files touched. Aggregation is **deterministic `jq`** via `scripts/build-insights.sh` (self-tested by `scripts/test-insights.sh`; neither counted by the doc-currency gate). **COST (tokens/$) is intentionally NOT captured** — that lives in Claude Code's own transcripts, so the dashboard points to `npx ccusage`. This is **+1 command (13 → 14)**; **no new agent / skill / hook (still 19 hooks)** and **no schema change**. Additive on top of v14.6.0.

**v14.6.0 — `/capability-check`: on-demand capability-drift scan against a tracked baseline:** Adds a new maintainer/self-evolution slash command `/capability-check` that performs an **on-demand, bounded (≤5 fetches)** scan — it diffs the live Claude Code changelog/docs plus dependency info against a tracked `ai-agent-manager-plugin/docs/CAPABILITY_BASELINE.json` and reports **CANDIDATE** adoptions (new platform capabilities the plugin could adopt). It **never self-applies** any change, **suppresses no-change output** (silent when nothing new), and `--update-baseline` is an explicit maintainer action (not run during a normal scan). This is **+1 command (12 → 13)**; **no new agent / skill / hook (still 19 hooks)** and **no new result schema**. The scheduled-scanner variant remains **deferred to P5**. Additive on top of v14.5.0.

**v14.5.0 — Memory consume-path: `/dreaming` collects, distills, and writes-on-approval:** Closes the consume-half of project memory that v14.4.0 deferred to P4. `/dreaming` now (a) **collects** worker `WORKER_RESULT.memory_candidates[]` from worker summaries / session logs / done+failed briefs, (b) **distills bounded `LESSONS.md`** — at most **≤3 active entries per category** (a Reflexion-style sliding window, scored, oldest-evicted by the new repo-root sole writer **`write-lessons.sh`**), and (c) **writes accepted memory/LESSONS on per-item approval** via the repo-root sole writers (`write-project-memory.sh` / `write-lessons.sh`). This evolves `/dreaming` from strict propose-only to **propose-and-write-on-approval**, but it stays **human-gated** — no auto-write, and `CLAUDE.md` + legacy agent-memory remain paste-to-apply. **APPLY (reading LESSONS back at plan time) and auto-demotion are deferred to P5; LESSONS provenance is deferred to P5.** This added two scripts (`write-lessons.sh` + its test, not counted by the doc-currency gate) but **no new command / agent / skill / hook — still 19 hooks**, and **WORKER_RESULT stays `schema_version` 2**. Additive on top of v14.4.0.

Additive on top of v14.3.0.

**v14.3.0 — Advisory project memory (P2b):** Adds agent-writable, cross-session **project memory** so the plugin stops re-discovering the codebase every run — built behind the red-team's guardrails. v1 layout under the gitignored `.supervisor/memory/`: `PROJECT_MEMORY.md` (advisory index, ≤200 lines) + `.provenance.jsonl` (hash-chained). (On-demand `topics/<topic>.md` detail files are a deferred P4 extension — v1 ships the index + provenance only.) **Sole writer** `scripts/write-project-memory.sh` **refuses any git-worktree CWD** (closes the F1 worktree-divergence FATAL — only repo-root callers write), hash-chains each entry, and enforces write-time eviction at the cap. **Read-side gate** `scripts/read-project-memory.sh` validates the chain and emits ONLY verified entries — an out-of-band poisoned line, or anything past a broken chain link, is dropped + logged to `.supervisor/logs/memory.log` (closes W1). Memory is **advisory and strictly subordinate to `CLAUDE.md`** (never an enforcement boundary), and **promotion is human-gated**: Launch Pad reads memory during Phase 3 ANALYZE and *proposes* new facts at Phase 6 for explicit user approval before any write. **v14.2.2 — Notification polish (patch):** Rounds out the v14.1.0 notification surface. `notify-desktop.sh` gains (1) **burst debounce** — coalesces rapid notification bursts (parallel hook fires / back-to-back gates) into one banner within a window set by `AI_AGENT_MANAGER_NOTIFY_DEBOUNCE` (default 5s; `0` disables; state in a gitignored `.supervisor/logs/.notify-debounce`), and (2) **Linux channel-detection** — only attempts `notify-send` when `DISPLAY`/`WAYLAND_DISPLAY` is set, skipping cleanly on headless boxes (the webhook is the headless channel). **v14.2.1 — Webhook + telemetry result-extraction fix (patch):** `send-webhook.sh` (the `supervisor_result` path) and `send-telemetry-core.sh` previously read the finishing subagent's output from a top-level **`result_block`** field that a real Claude Code **`SubagentStop`** payload never carries (verified by capturing a live payload). They now use the same robust resolution chain as `scripts/validate-launch-pad-result.py`: **`last_assistant_message`** (the real, observed inline field) → legacy `result_block` / `output` / `agent_output` (back-compat) → the last assistant message read from the transcript JSONL, preferring the subagent-scoped **`agent_transcript_path`** then the shared **`transcript_path`**. The pre-fix readers resolved `.result_block` to empty, so the supervisor-completion webhook (shipped v14.1.0) and every telemetry post were silently suppressed (status empty → POST skipped / `no_known_result_block`). The telemetry core also raw-scans the *resolved* result text for secrets before redaction (transcript-sourced text is not a payload field, so it would otherwise bypass the fail-closed privacy gate). `scripts/validate-launch-pad-result.py` (already on the `last_assistant_message` chain since v14.2.0) had its transcript fallback aligned to prefer the subagent-scoped `agent_transcript_path` too, so all three SubagentStop readers stay in sync. New dry-run self-tests: `scripts/test-webhook.sh` (real-field / legacy / transcript-fallback / suppression cases) and two discriminating `*-lastmsg-*` telemetry fixtures wired into `scripts/test-telemetry.sh`. Authoritative payload shape + resolution chain documented in `ai-agent-manager-plugin/docs/TELEMETRY.md` §"Result-text extraction". **No** hook-count (still 19), schema, agent, command, or skill change. Additive patch on top of v14.2.0.

**v14.0.0 — Continuous autonomous mode with stacked PRs:** The `/autonomous` orchestration shell graduates from v13's opt-in re-planning to a **default multi-iteration loop** (cap 10, default 3) with **stacked-branch semantics**: iteration N+1 branches from `iterations[N].branch` rather than the integration base, so successive iterations form a reviewable stack. `--single-iteration` (or `--max-iterations 1`) preserves v13 single-PR behavior; `--no-stacked-branches` restores v13's branch-from-main cadence while keeping multi-iter mode active. New flags: `--notify` enables opt-in gate-event webhooks (rubric gate, adjudication gate, NO-GO override, Plan Review FAIL × 3) gated on `AI_AGENT_MANAGER_WEBHOOK_URL`; `--non-interactive-fallback` enables a per-gate fail-closed policy for CI / stdin-not-tty environments (rubric gate aborts with `rubric_gate_closed_non_interactive`; no-rubric `completed` runs return `done` with `no_rubric_in_non_interactive`; adjudication gate inherits Supervisor's `--non-interactive` policy if forwarded). Supervisor gains `--base-branch <ref>` and `--non-interactive` flags plus Phase 0/4/4.5 base-mismatch detection + cleanup, and emits new optional additive `branch_base` + `pr_state` fields on `SUPERVISOR_RESULT` (schema_version remains 1, both fields are optional). Context-Keeper gains atomic `set_flag` / `get_flag` / `clear_flag` operations writing under a new `## Phase Flags` section in `state.md` (consumed by autonomous-loop for stacked-branch handoff). `AUTONOMOUS_RUN` bumps to **schema_version 2** with nine new closed `status_reason` values covering the stacked-branch / non-interactive-fallback / gh-retry / no-rubric / deprecated-flag-conflict paths — v1 artifacts still accepted for the transition window. The authoritative list (with status-pairing) lives in `ai-agent-manager-plugin/docs/RESULT_SCHEMAS.md` §"AUTONOMOUS_RUN"; key examples include `non_interactive_without_fallback`, `iter_pr_base_mismatch`, `rubric_gate_closed_non_interactive`, `no_rubric_in_non_interactive`, `user_aborted_gh_retry`, `supervisor_base_branch_mismatch`. **W-NEW-3 spike PASSED pre-merge:** Code Reviewer and Rubric Grader both honor `DIFF-SCOPE OVERRIDE` inline directives on stacked-branch fixtures, removing the architectural blocker. Gate-event webhook payloads use **jq-only injection-safe construction** (no shell interpolation into JSON). v14 is **additive on top of v13**: no new agent, no hook count change (still 14), no removal of any v12.x / v13 capability. All v13.0.0 / v13.0.1 capabilities preserved (foreground-assisted gates bubble `AskUserQuestion` in-session; rubric-gate user-merge verification; `failed + inter_subtask_gap` Option C re-plan; webhook empty-payload suppression).
