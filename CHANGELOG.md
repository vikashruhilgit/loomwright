**v14.8.0 — Phase 1.5 PRE-FLIGHT SYNC: remote-state reconciliation before any worker spawns:** Adds a mandatory **Phase 1.5 PRE-FLIGHT SYNC** gate to the Supervisor, placed *after* Phase 1 ACQUIRE (which already fetches/pulls the base so the feature branch starts fresh) and *before* Phase 2 PLAN spawns the Orchestrator or any worker — so overlapping or already-landed work is surfaced *before* tokens are spent on decomposition/execution.
